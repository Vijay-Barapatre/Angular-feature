ğŸ¯ Key Concepts

Initialization (ngOnInit)

Called once after the first ngOnChanges. Perfect for fetching data and setting up logic.

ngOnInit(): void { this.loadData(); }

Cleanup (ngOnDestroy)

Called just before Angular destroys the component. Unsubscribe and release resources here.

ngOnDestroy(): void { this.sub.unsubscribe(); }

Input Changes (ngOnChanges)

Called before ngOnInit and whenever input properties change. Receives SimpleChanges object.

ngOnChanges(changes: SimpleChanges): void

View Ready (ngAfterViewInit)

Called once after the component's view and child views are initialized.

ngAfterViewInit(): void { this.chart.init(); }
